### 상속 (Inheritance)
**상속**이란 기존 클래스(부모 클래스, 상위 클래스, 기반 클래스)의 **속성과 메서드를 새로운 클래스(자식 클래스, 하위 클래스, 파생 클래스)** 가 **재사용**하거나 **확장**할 수 있도록 하는 객체지향 개념

| 용어                                  | 의미                       |
| ----------------------------------- | ------------------------ |
| 부모 클래스 (Super Class / Parent Class) | 공통 속성과 기능을 정의한 기존 클래스    |
| 자식 클래스 (Sub Class / Child Class)    | 부모 클래스를 확장하여 새롭게 정의한 클래스 |
``` Java
class Parent {
    int a;
    void printA() { ... }
}

class Child extends Parent {
    int b;
    void printB() { ... }
}
```

### 상속의 목적
- **코드 재사용성 향상**: 중복 코드를 줄이고 유지보수 용이
- **계층적 구조 구성**: 일반 → 구체 개념으로 확장 가능
- **다형성 구현 기반 제공**: 부모 타입으로 자식 객체 참조 가능

---

### 상속의 특징
- Java는 **단일 상속**만 지원 (여러 클래스 상속 불가 → 인터페이스로 보완)
- 자식 클래스는 부모 클래스의 **public / protected 멤버**를 **자동 상속**
- 자식 클래스는 **필요시 오버라이딩**(재정의) 가능

---

### 상속의 단점 및 주의점
- **강한 결합 발생 가능**: 부모 클래스 변경 시 자식 클래스 영향
- **유연성 저하**: 잘못된 상속 구조는 유지보수성을 떨어뜨림
- **"is-a" 관계 검토 필수**: 단순 코드 재사용 목적이라면 **구성(Composition)** 이 더 적합할 수 있음
``` Java
// 강제로 상속하면 불필요한 기능도 상속됨
class Bird {
    void fly() {}
}
class Ostrich extends Bird {
    // 타당한 "is-a" 관계가 아님 → 구성(Has-a) 고려 필요
}
```
