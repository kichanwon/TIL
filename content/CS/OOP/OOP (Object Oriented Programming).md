# 객체지향이란 
> 현실 세계의 개념을 프로그램에 반영하여 **객체(속성과 동작을 지닌 독립 단위)** 간의 상호작용으로 문제를 해결하는 프로그래밍 패러다임

## 목적
- 높은 **응집도(High Cohesion)**: 관련 데이터와 기능을 클래스 내부로 모아 관리
- 낮은 **결합도(Low Coupling)**: 클래스 간의 의존성을 줄여, 유연한 유지보수 가능
- 유지보수, 확장, 재사용성 향상

---
## 기본 구성 요소
### 클래스(Class)
같은 종류(또는 문제 해결을 위한)의 집단에 속하는 
**속성(attribute)** 과 **행위(behavior)** 를 정의한 설계도로, 
객체를 생성하기 위한 **청사진 또는 틀**이다.  
다른 클래스나 외부 요소와 **독립적으로 설계**되어야 유지보수성과 재사용성이 높다.

### 객체(Object)
클래스로부터 생성된 **실체(instance)**로, 메모리에 할당되어 동작하는 단위이다.  
자신 고유의 속성을 가지며, 클래스에 정의된 행위를 공유함으로써 **메모리 효율성**을 갖는다.

### 메서드(Method) / 메시지(Message)
- **메서드**: 객체의 행위를 구현한 함수로, 객체의 속성값을 조작하거나 동작을 수행한다.
- **메시지**: 객체에 특정 메서드를 수행하도록 **요청하는 명령 또는 호출**이다.
> 객체 간의 통신은 **메시지 전달 → 메서드 호출** 방식으로 이루어진다.

## 객체와 인스턴스 차이
- 클래스에 의해 만들어진 **모든 객체는 인스턴스**이다.
- `Animal cat = new Animal();`  
    → `cat`은 객체이며 동시에 **Animal 클래스의 인스턴스**이다.

> *"객체"* 는 메모리에 존재하는 실체를 의미하고, 
> *"인스턴스"* 는 어떤 클래스에 속해 있다는 **관계 중심의 표현**이다.  
> 예: "cat은 Animal의 객체(X) → 인스턴스(O)"

## 객체란 무엇인가?
- **속성(데이터)** 과 **기능(동작)** 이 있는, 다른 것들과 구분 가능한 독립된 존재 
- 현실 세계는 객체의 집합이며, 사건은 객체 간의 상호작용으로 설명 가능
`예: 내가 문을 열었다 → 나(object)가 문(object)을 열었다(function)`

## 객체지향 설계 5원칙 ([[SOLID]])
> 유지보수성과 확장성을 높이기 위한 **객체지향 설계 원칙 5가지**

| 원칙                       | 설명 요약                                                  |
| ------------------------ | ------------------------------------------------------ |
| **S: 단일 책임 원칙 (SRP)**    | 클래스는 하나의 책임만 가져야 하며, 변경 이유도 하나여야 한다.                   |
| **O: 개방-폐쇄 원칙 (OCP)**    | 확장에는 열려 있고, 변경에는 닫혀 있어야 한다.                            |
| **L: 리스코프 치환 원칙 (LSP)**  | 부모 클래스를 사용하는 곳에 자식 클래스를 넣어도 문제가 없어야 한다.                |
| **I: 인터페이스 분리 원칙 (ISP)** | 하나의 인터페이스는 하나의 역할만 가지며, 클라이언트는 사용하지 않는 메서드에 의존하면 안 된다. |
| **D: 의존 역전 원칙 (DIP)**    | 고수준 모듈은 저수준 모듈에 의존하면 안 되며, 둘 다 추상화에 의존해야 한다.           |

## Java 메모리
### Java 메모리 구조와 변수 분류

| 메모리 영역                  | 저장되는 내용                     | 대표 변수 예시              |
| ----------------------- | --------------------------- | --------------------- |
| **Method Area**         | 클래스 정보, static 변수, 메서드 코드 등 | static 변수, static 메서드 |
| **Heap Area**           | 객체(instance)와 인스턴스 변수       | new로 생성한 객체, 인스턴스 필드  |
| **Stack Area**          | 메서드 호출 시 생성되는 지역 변수, 매개변수   | 지역 변수, 매개변수           |
| **PC Register**         | 각 스레드가 실행 중인 명령어 주소         | (개발자가 직접 다루지 않음)      |
| **Native Method Stack** | JNI 호출 관련 스택                | (개발자가 직접 다루지 않음)      |

### 변수 분류 정리

| 변수 유형         | 메모리 위치      | 소멸 시점                | 특징                       |
| ------------- | ----------- | -------------------- | ------------------------ |
| **static 변수** | Method Area | 클래스 언로드 시            | 모든 인스턴스에서 **공유**됨        |
| **인스턴스 변수**   | Heap        | 객체 소멸 시              | 객체마다 별도 존재               |
| **지역 변수**     | Stack       | 메서드 종료 시 자동 소멸       | 초기화하지 않으면 사용 불가 (컴파일 오류) |
| **매개변수**      | Stack       | 메서드 호출 시 생성, 종료 시 소멸 | 지역 변수와 동일하게 동작           |
```java
class Counter {
    static int count = 0; // 모든 객체가 공유

    public Counter() {
        count++;
    }
}
```
