### 캡슐화 (Encapsulation)
**캡슐화**란 객체의 **데이터(속성)** 와 **기능(메서드)** 을 **하나의 단위(클래스)** 로 묶고,  
**외부에는 내부 구현을 숨기며, 공개된 인터페이스를 통해서만 접근하도록 제한**하는 객체지향 개념

### 정보 은닉과 접근 제어자
> 내부 구현 감춤(정보 은닉)을 위해 **접근 제어자**를 사용, **컴파일 시 유효성 검사**
> 접근 범위 위반 시 컴파일 오류 발생 → 실행 코드 생성 불가

---
### 클래스 수준 접근 제어자 (Java 기준)

| 접근 제어자    | 접근 범위 설명                           |
| --------- | ---------------------------------- |
| `public`  | 다른 패키지 포함 모든 클래스에서 접근 가능           |
| (default) | 동일 패키지 내 클래스에서만 접근 가능 (명시하지 않은 경우) |
\* `private`, `protected`는 클래스 선언에 사용 불가

---
### 멤버(필드/메서드) 접근 제어자

| 제어자         | 접근 범위                                | 상속  |
| ----------- | ------------------------------------ | --- |
| `private`   | **해당 클래스 내부**에서만 접근 가능 (외부/상속 모두 불가) |     |
| (default)   | 동일 패키지 내에서만 접근 가능                    |     |
| `protected` | 동일 패키지 + **상속받은 클래스**에서 접근 가능        |     |
| `public`    | 모든 외부 클래스에서 접근 가능                    |     |

---
#### 예시 정리
``` Java
public class Member {
    private String name;         // 외부 접근 불가 → getter/setter 통해 접근
    protected int age;           // 같은 패키지 + 자식 클래스에서 접근 가능
    public void printInfo() {}   // 어디서든 호출 가능
}
```
- `private`: 캡슐화의 **핵심 구현 수단**, 필수 속성에 직접 접근 제한
- `public`: **외부 노출 인터페이스**로 제한된 방식으로 접근 허용
