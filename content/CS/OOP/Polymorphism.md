### 다형성 (Polymorphism)
**하나의 이름(인터페이스, 메서드 등)** 이 **다양한 동작(행위)** 을 갖도록 하는 객체지향의 핵심 개념
> **"하나의 코드가 다양한 타입을 처리할 수 있게 함"** 으로써, **유연하고 확장 가능한 구조 설계**가 가능

| 구분                                | 설명          | 예시                                  |
| --------------------------------- | ----------- | ----------------------------------- |
| **정적 다형성** (Static Polymorphism)  | 컴파일 시점에 결정됨 | **오버로딩** (Overloading)              |
| **동적 다형성** (Dynamic Polymorphism) | 실행 시점에 결정됨  | **오버라이딩** (Overriding), 인터페이스 다형성 등 |

### 오버로딩 (Overloading)
- **같은 이름**의 메서드를 **매개변수의 타입, 개수, 순서**에 따라 **여러 개 정의**하는 것
- 컴파일 타임에 호출될 메서드가 정해짐 → **정적 바인딩**
``` Java
void print(String s) {}
void print(int i) {} // 오버로딩`
```
---

### 오버라이딩 (Overriding)
- **부모 클래스의 메서드를 자식 클래스에서 재정의**하는 것
- 런타임에 어떤 메서드가 실행될지 결정됨 → **동적 바인딩**
- 조건: 메서드 이름, 매개변수, 반환형이 **완전히 동일**해야 함
``` Java
class Animal {
  void sound() { System.out.println("Some sound"); }
}
class Dog extends Animal {
  @Override
  void sound() { System.out.println("Bark"); } // 오버라이딩
}```
---

### 인터페이스 기반 다형성
``` Java
void execute(Worker w) {
  w.run();
}
// Worker 인터페이스를 구현한 객체는 어떤 것이든 전달 가능
```

- 부모 타입의 참조 변수로 **여러 자식 객체를 대체할 수 있는 구조**
- **전략 패턴**, **의존성 주입**, **동적 바인딩** 등에 활용

---

### 다형성의 장점
- **코드 재사용성**과 **유지보수성 향상**
- **의존성 최소화** 및 **인터페이스 기반 설계 유도**
- 다양한 객체의 동작을 동일한 인터페이스로 다룰 수 있음
