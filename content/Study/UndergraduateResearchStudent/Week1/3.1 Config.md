|BATCH_SIZE|GPU 메모리|학습 속도|정확도|권장 상황|
|---|---|---|---|---|

**주의사항:**

### 4.2 Gradient Accumulation (ACCUMULATION_STEPS)
```python
ACCUMULATION_STEPS = 4
```
#### 개념
**Gradient Accumulation (경사 누적):** 작은 배치의 gradient를 여러 번 누적한 후 한 번에 가중치를 업데이트하는 기법
#### 동작 원리
```python
# 일반적인 학습 (ACCUMULATION_STEPS = 1)
for batch in dataloader:
    loss = model(batch)
    loss.backward()      # gradient 계산
    optimizer.step()     # 즉시 업데이트
    optimizer.zero_grad()

# Gradient Accumulation (ACCUMULATION_STEPS = 4)
for i, batch in enumerate(dataloader):
    loss = model(batch)
    loss.backward()      # gradient 누적
    
    if (i + 1) % 4 == 0:
        optimizer.step()     # 4번마다 업데이트
        optimizer.zero_grad()
```
#### Effective Batch Size
```python
BATCH_SIZE = 128
ACCUMULATION_STEPS = 4

# 실제 GPU에 올라가는 배치: 128
# Gradient 누적 후 효과적인 배치: 128 × 4 = 512
Effective Batch Size = 128 × 4 = 512
```
---
#### 장단점
**장점:**
1. **메모리 절약**: 큰 배치를 직접 올리지 않아도 됨
2. **큰 배치 효과**: gradient 안정화, 노이즈 감소
3. **GPU 메모리 한계 극복**: 작은 GPU에서도 큰 배치 효과
**단점:**
4. **학습 속도 저하**: 업데이트 주기가 늦어짐
5. **일반화 성능**: 너무 큰 effective batch는 과적합 위험
6. **하이퍼파라미터 조정 필요**: 학습률 스케일링 필요
---
#### 실험 결과 분석
**ACCUMULATION_STEPS = 8 (큰 누적)**
```
BATCH_SIZE | Effective | GPU 사용 | 정확도 | 시간
64         | 512      | 581.2    | 0.586  | 2m 37s
128        | 1024     | 618.0    | 0.599  | 2m 28s  ← 최적
256        | 2048     | 692.0    | 0.552  | 2m 34s
512        | 4096     | 837.6    | 0.509  | 2m 52s  ← 과도함
```
**ACCUMULATION_STEPS = 1 (누적 없음)**
```
BATCH_SIZE | Effective | GPU 사용 | 정확도 | 시간
128        | 128      | 441.9    | 0.634  | 3m 2s   ← 최고 정확도!
512        | 512      | 661.9    | 0.596  | 2m 52s
```
**결론:**
- **정확도 우선**: `ACCUMULATION_STEPS = 1`, `BATCH_SIZE = 128` → **0.634**
- **속도 우선**: `ACCUMULATION_STEPS = 8`, `BATCH_SIZE = 128` → 2m 28s
- **균형**: `ACCUMULATION_STEPS = 4`, `BATCH_SIZE = 128` → 속도와 정확도 타협
---
#### 학습률 스케일링
```python
# ACCUMULATION_STEPS = 1
LEARNING_RATE = 0.001

# ACCUMULATION_STEPS = 4
LEARNING_RATE = 0.001 * 4  # 학습률도 같이 증가!
```
**이유:**
- Effective Batch Size가 커지면 gradient가 평균화되어 작아짐
- 학습률을 비례적으로 증가시켜 보상
- 일반적 규칙: `LR_new = LR_base × ACCUMULATION_STEPS`
**실험 결과:**
```
Config                              | 정확도 | 시간
ACCUMULATION=1, LR=0.001           | 0.631  | 2m 56s
ACCUMULATION=4, LR=0.001*4 (0.004) | 0.578  | 2m 45s
```

### 4.3 에포크 수 (NUM_EPOCHS)
```python
NUM_EPOCHS = 3
```

|항목|설명|
|---|---|
|**의미**|전체 데이터셋을 몇 번 반복 학습할지|
|**영향**|많을수록 학습 충분, 과적합 위험|
|**선택 기준**|검증 손실이 더 이상 개선되지 않을 때까지|
**에포크별 학습 과정:**
```
Epoch 1: 초기 학습 (큰 폭 개선)
Epoch 2: 추가 학습 (중간 개선)
Epoch 3: 미세 조정 (작은 개선)
Epoch 4+: 과적합 위험 (검증 손실 상승)
```
**적정 에포크 찾기:**
- **너무 적으면**: 미학습 (underfitting)
- **너무 많으면**: 과적합 (overfitting)
- **최적값**: Early Stopping 사용 권장

### 4.4 학습률 (LEARNING_RATE)
```python
LEARNING_RATE = 0.001 * 4  # 0.004
```

|항목|설명|
|---|---|
|**의미**|가중치 업데이트 스텝 크기|
|**영향**|클수록 빠르게 학습, 불안정 위험|
|**일반 범위**|0.0001 ~ 0.01|
**학습률 영향:**
```
LR 너무 큼 (0.1)    → 발산 (loss가 NaN)
LR 적당함 (0.001)   → 안정적 학습 ✅
LR 너무 작음 (0.00001) → 학습 느림, 미학습
```
**학습률 시각화:**
```
Loss
 │
 │  LR=0.1 (너무 큼)
 │     ╱╲╱╲╱╲  발산!
 │    ╱      ╲
 │───────────────
 │
 │  LR=0.001 (적절)
 │    ╲___    수렴
 │        ╲___
 │            ╲___
 │
 │  LR=0.0001 (너무 작음)
 │    ╲       느린 수렴
 │     ╲
 │      ╲
 └─────────────────→ Epoch
```
**학습률 스케줄링:**
```python
# Cosine Annealing
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=NUM_EPOCHS
)

# Step Decay
scheduler = torch.optim.lr_scheduler.StepLR(
    optimizer, step_size=10, gamma=0.1
)
```

### 4.5 모멘텀 (MOMENTUM)
```python
MOMENTUM = 0.9
```

|항목|설명|
|---|---|
|**의미**|과거 gradient의 관성 유지 정도|
|**범위**|0.0 ~ 1.0 (일반적으로 0.9)|
|**효과**|학습 안정화, 지역 최솟값 탈출|
**모멘텀 개념:**
```python
# 모멘텀 없음 (momentum = 0)
velocity = 0
weight -= learning_rate * gradient

# 모멘텀 있음 (momentum = 0.9)
velocity = 0.9 * velocity + gradient
weight -= learning_rate * velocity
```
**물리적 비유:**
- 모멘텀 = 공이 굴러가는 관성
- 과거 방향을 기억하여 지그재그 움직임 완화
- 언덕을 더 빠르게 내려감
**모멘텀 효과:**
```
모멘텀 없음 (0.0)     모멘텀 있음 (0.9)
    ╱╲╱╲╱╲              ╲
   ╱      ╲              ╲___
  ╱        ╲                 ╲___
 ╱지그재그   ╲                   ╲ 부드럽게 수렴
```

## 5. 데이터 분할 설정
```python
TEST_SIZE = 0.1
VAL_SIZE = 0.2
RANDOM_STATE = 42
```
### 5.1 분할 비율

|변수|비율|설명|
|---|---|---|
|`TEST_SIZE`|10%|최종 평가용 데이터|
|`VAL_SIZE`|20%|학습 중 검증용 데이터|
|Train|70%|실제 학습 데이터|
**전체 데이터셋 (25,000장) 분할:**
```
전체 25,000장
    ↓
├─ Test:  2,500장 (10%) → 최종 평가
└─ 나머지 22,500장
    ↓
    ├─ Val:   4,500장 (20%) → 학습 중 검증
    └─ Train: 18,000장 (80%) → 실제 학습
```
**분할 이유:**
- **Train**: 모델 학습
- **Val**: 과적합 체크, 하이퍼파라미터 튜닝
- **Test**: 최종 성능 평가 (한 번만 사용!)
---
### 5.2 랜덤 시드 (RANDOM_STATE)
```python
RANDOM_STATE = 42
```

|항목|설명|
|---|---|
|**역할**|데이터 분할 재현성 보장|
|**값**|아무 정수 (관습적으로 42)|
|**효과**|실험 재현 가능, 공정한 비교|
**재현성 예시:**
```python
# RANDOM_STATE = 42 (고정)
train1, val1 = split_data(data, random_state=42)
train2, val2 = split_data(data, random_state=42)
# train1 == train2 ✅ (동일한 분할)

# RANDOM_STATE = None (랜덤)
train1, val1 = split_data(data, random_state=None)
train2, val2 = split_data(data, random_state=None)
# train1 != train2 ❌ (다른 분할)
```

## 핵심 포인트 정리
### Config의 중요성
1. **중앙 관리**: 모든 설정을 한 곳에서 관리
2. **재현성**: RANDOM_STATE로 실험 재현
3. **유지보수**: 값 변경이 쉽고 명확
### 하이퍼파라미터 선택
- **학습률**: 가장 중요! 0.001부터 시작
- **배치 크기**: GPU 메모리 한계 내에서 최대
- **에포크**: Early Stopping 권장
### Gradient Accumulation
- 큰 배치 효과를 메모리 절약하며 구현
- 학습률 스케일링 필수
- 정확도와 속도 트레이드오프 존재
---
