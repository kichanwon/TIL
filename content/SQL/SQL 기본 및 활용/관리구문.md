| **데이터베이스 객체**   | **설명**                                       |
| --------------- | -------------------------------------------- |
| 테이블(Table)      | 데이터를 저장하는 객체                                 |
| 인덱스(Index)      | 테이블의 검색 효율을 높이기 위해 사용하는 객체                   |
| 뷰(View)         | 하나 이상의 데이터를 논리적으로 연결하여 하나의 테이블처럼 사용하게 해주는 객체 |
| 시퀀스(Sequence)   | 일련 번호를 생성해주는 객체                              |
| 시노님(Synonym)    | 데이터베이스 객체에 대한 별칭을 부여하는 객체                    |
| 프로시저(Procedure) | 프로그래밍 연산 및 기능을 수행하고 값을 반환하지 않는 객체            |
| 함수(Function)    | 프로그래밍 연상 및 기능을 수행하고 값을 반환하는 객체               |
| 패키지(Package)    | 관련 있는 프로시저와 함수를 하나로 묶어 놓은 객체                 |
| 트리거(Trigger)    | 데이터 관련 작업의 연결 및 방지 관련 기능을 제공하는 객체            |

시퀀스, 테이블, 인덱스 모두 다 별도의 객체(Object)
각각이 유기적으로 동작하긴 하지만 각각의 특성은 다름
  
시퀀스는 단순하게 유니크한 새 번호를 따기 위한 구조
시퀀스는 테이블에 종속적이지 않음
테이블 입력시 사용되어질 뿐

테이블은 데이터를 저장하기 위한 구조
저장이 주 목적이며, 정렬은 고려하지 않음

인덱스는 테이블의 데이터를 빨리 찾기 위한 구조
데이터를 빨리 찾기 위해 정렬된 구조를 가지고 있음
인덱스는 테이블에 종속적

### DDL
>Data Definition Language : 데이터 정의 언어
>객체(OBJECT)를 CREATE, ALTER, DROP 

#### Create
	테이블 생성
- 객체를 의미하는 것이므로 단수형으로 이름을 짓는걸 권고한다.
- 유일한 이름으로 명명해야 한다.
- 테이블 내의 컬럼명 또한 중복되지 않는 유일한 이름으로 명명해야 한다.
- 정의할 때 각 컬럼은 `,`으로 구분하며 테이블 생성문의 마지막은 `;`이다.
- 컬럼명은 데이터 표준화 관점에서 일관성 있게 사용해야 한다.
- 컬럼 뒤에 데이터 유형을 반드시 지정해야 한다.
- 테이블과 컬럼명은 반드시 문자로 시작한다.
- 대소문자 구분을 하지 않지만, 기본적으로 대문자로 만들어진다.
#### Alter
	테이블의 구조 수정

| 명령어                | 내용                     |
| ------------------ | ---------------------- |
| **ADD COLUMN**     | 컬럼을 추가하는 역할            |
| **DROP COLUMN**    | 컬럼을 삭제하는 역할            |
| **MODIFY COLUMN**  | 컬럼을 수정하는 역할            |
| **RENAME COLUMN**  | 컬럼 이름을 변경하는 역할         |
| **DROP CONSTRAIN** | 컬럼을 제약조건을 기반해서 삭제하는 역할 |
#### Drop
	테이블 삭제
#### Rename
	테이블 이름 변경
#### Comment
	테이블 컬럼명 주석
#### Truncate
	테이블 초기화

### **_제약 조건_**

| **Constraints**      | **NULL 허용 여부**                                                                  | **데이터 중복 허용여부**                       | **특징**                                        |
| -------------------- | ------------------------------------------------------------------------------- | ------------------------------------- | --------------------------------------------- |
| **NOT NULL**         | NULL 불가                                                                         | 중복 가능                                 |                                               |
| **UNIQUE**           | NULL 가능                                                                         | 중복 불가<br><br>(* NULL끼리는 중복으로 간주하지 않음) |                                               |
| **PRIMARY KEY(고유키)** | NULL 불가                                                                         | 중복 불가                                 | 지정한 열은 유일한 값을 반드시 가져야 함<br><br>테이블 당 1개만 지정가능 |
| **FOREIGN KEY(외래키)** | 다른 테이블 열을 참조하여<br><br>해당 테이블에 존재하는 값만 입력 가능<br><br>다른 테이블의 고유키(PRIMARY KEY)를 참조 |                                       |                                               |
| **CHECK**            | 설정한 조건식을 만족하는 데이터만 입력가능<br><br>조건식을 만족하지 않는 데이터는 입력이 거부됨                        |                                       |                                               |
